## What this project is

`build.py` from `content/*.md`, served from `docs/` via GitHub Pages at
`https://www.evanmayo-wilson.org`. No test suite, no linter, no CI beyond
GitHub Pages' own build-and-deploy and two scheduled Actions
(`scholar-stats.yml` weekly, `pub-stats.yml` daily). Workflow is direct
commits to `main` — there is no branching model, no PRs, no session branches.

3. **New features**: live search box in the publications toolbar (small

   (`Author_Year_Title_words.ext`).
4. **Two real PDF-resolution bugs found and fixed** in
   `generate_pub_stats.py`, both via Evan manually clicking buttons and
   reporting bad behavior:
   - A publisher's `citation_pdf_url` meta tag was only checked against the
     DOI-redirect landing page, not every OA location — missed some
     legitimately-open articles.
   - **The big one**: Unpaywall's `url_for_pdf` field was trusted with zero
     verification, unlike every other candidate in the resolution chain. A
     JAMA Internal Medicine article's PDF button landed on a Cloudflare
     bot-challenge HTML page instead of a PDF. Fix: every candidate,
     including `url_for_pdf`, now gets a Range request checked for the
     `%PDF-` magic number before being trusted. Re-running dropped the
     "resolved" count from 102 to 76 — meaning 26 buttons across the site had
     silently been pointing at non-PDFs. Not a hypothetical; see
     `CLAUDE.md`'s Gotchas section.

   legitimate live-resolvable PDF (mostly ones stuck behind SAGE/JAMA/BMJ/

## Gotchas

See `CLAUDE.md`'s own "Gotchas" section — kept there rather than duplicated
here since that's the file every session reads first. Highlights: verify
every PDF candidate including `url_for_pdf` (real bug, already fixed once);
CSS `font:` shorthand resets `line-height` if declared after it; the
site-wide `[hidden] { display: none !important; }` rule needs `!important`
overrides to keep a hidden element in-flow; SQLite `LIKE` treats `_` as a
wildcard; a `function` declared inside one `if`/`else` branch isn't reliably
hoisted for use before the block. No test suite/linter exists — "verify"
means rebuild cleanly, `node --check` the extracted inline `<script>`, and
actually load the page in a browser.
